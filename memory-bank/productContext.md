# Product Context — GalacticExpansion (GLEX)

## Зачем существует проект

GLEX нужен, чтобы на dedicated сервере Empyrion: Galactic Survival появился “живой” противник Zirax на уровне планеты:

- высадка игрока сопровождается патрулями/перехватом;
- Zirax строят и улучшают базу по стадиям;
- экспансия и “экономика” реализуются как симуляция;
- действия игроков (разрушение, зачистка) вызывают реакцию (усиление, перестройка, откат стадии).

## Как это должно ощущаться (UX)

- Игроки видят “следы жизни”: логистический корабль, появление стройплощадки, рост базы.
- Угроза появляется не хаотично, а по понятным правилам (присутствие игрока, этап колонии, разрушения).
- Поведение мода предсказуемо и диагностируемо по логам.

## Границы (scope)

- Server-side DLL мод, без клиентской установки.
- Опора на публичные возможности ModAPI: события/запросы, спавн/удаление сущностей, консольные команды (AIM).

## Не-цели (non-goals) на ближайшие версии

- Полноценное управление AI-маршрутами кораблей, если API этого не поддерживает.
- Глубокая симуляция блокового урона (если ModAPI ограничена уровнем entity).
