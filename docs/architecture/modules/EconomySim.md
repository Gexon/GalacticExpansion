# Модуль `EconomySim`

**Ответственность**: симуляция ресурсов/производства/логистики Zirax (виртуальные ресурсы, скорости, затраты на стадии).  
**Аудитория**: разработчики.  
**Статус**: draft.  
**Связанные документы**: [`../../design/01_gameplay_simulation.md`](../../design/01_gameplay_simulation.md).  

## Идея

В MVP “добыча” — это **не чтение реальных депозитов**, а расчёт по правилам:

- `resources += productionRate * dt`
- затраты на апгрейд стадии списываются из виртуального банка

## Входы

- Текущая стадия колонии.
- Параметры из конфига (productionRate, costs, лимиты аутпостов).

## Выходы

- Обновления “виртуального банка” ресурсов в state.
- Сигналы “можно апгрейдить стадию”.
