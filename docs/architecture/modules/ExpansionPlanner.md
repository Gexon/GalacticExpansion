# Модуль `ExpansionPlanner`

**Ответственность**: планирование экспансии на новые планеты (когда колония достигла `Base_Max`).  
**Аудитория**: разработчики.  
**Статус**: draft.  

## Входы

- Состояние колоний (`state.json`) и их стадии.
- Конфиг экспансии (разрешено/запрещено в MVP, задержки, правила выбора цели).

## Выходы

- План “создать новую колонию” на целевом playfield.
- Условия выполнения: “есть игроки/плейфилд загружен” или отложенное выполнение.

## Ограничения

В требованиях отмечено: если API не даёт “маршрута”, перелёт реализуется таймером и появлением на целевом playfield при выполнении условий.
