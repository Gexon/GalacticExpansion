# Модуль `PlayerTracker`

**Ответственность**: отслеживание игроков и их присутствия на playfield’ах (в первую очередь `Event_Player_ChangedPlayfield`).  
**Аудитория**: разработчики.  
**Статус**: draft.  

## Зачем нужен

Для GLEX критично понимать: “есть ли игроки на целевом playfield сейчас”. Это влияет на:

- можно ли выполнять “видимые” действия (логистика/стройка/угрозы),
- стоит ли откладывать действия до момента загрузки playfield.

## Входы

- `Event_Player_ChangedPlayfield`
- `Event_Player_Connected` / `Event_Player_Disconnected` (если используем)

## Выходы

- “карта присутствия” игроков по playfield’ам (для остальных модулей).
