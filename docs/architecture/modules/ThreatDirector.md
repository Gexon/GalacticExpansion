# Модуль `ThreatDirector`

**Ответственность**: решает, когда и какие угрозы создавать (патрули/волны/дронбаза) в ответ на присутствие игроков и события.  
**Аудитория**: разработчики.  
**Статус**: draft.  
**Связанные документы**: [`../../design/03_threat_director.md`](../../design/03_threat_director.md).  

## Входы

- Контекст игроков (`PlayerTracker`).
- Контекст колоний/стадий (`state.json`).
- События разрушений (`StructureTracker`).

## Выходы

- Планы на AIM (через `AimOrchestrator`).
- Обновления state (например “повысили уровень тревоги”).

## Инварианты

- Угрозы ограничены лимитами (`MaxDroneWavesPerHour` и т.п.).
- Угрозы не должны “спавниться в вакууме” — без условий и без логирования причины.
