# ADR-0003: Whitelist и rate limit для `Request_ConsoleCommand`

**Статус**: accepted (по NFR из черновика).  
**Контекст**: GLEX использует консольные команды для AIM (`aim ...`). Это потенциально опасная поверхность (abuse/спам/нестабильность).  

## Решение

- Все вызовы `Request_ConsoleCommand` идут через единый модуль (`AimOrchestrator`).
- Разрешаем только команды из whitelist.
- Вводим `MaxAIMCommandsPerMinute` (и логируем блокировки).

## Причины

- безопасность и предсказуемость поведения;
- защита от лагов и “заспама” сервера;
- упрощение отладки (все консольные команды централизованы).

## Последствия

- часть “удобных” команд придётся явно добавлять в whitelist (осознанно).
