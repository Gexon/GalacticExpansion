# ADR-0001: Переход стадий через destroy+spawn prefab

**Статус**: accepted (по текущему черновику требований).  
**Контекст**: GLEX должен развивать базу по стадиям (`Base_L1 → Base_L2 → ...`) и реагировать на разрушение. ModAPI надёжнее работает на уровне сущностей, а не “поштучных блоков”.  

## Решение

Для смены стадии используем надёжный паттерн:

- **destroy** текущей структуры (entity уровня),
- **spawn** новой структуры с нужным `prefabName`.

## Причины

- проще обеспечить воспроизводимость;
- меньше зависимостей от “тонкостей” блоковой модели;
- легче откатывать стадии при разрушениях.

## Последствия

- требуется аккуратно вести ссылки на `entityId` в `state.json`;
- визуально может быть “прыжок” (исчез/появился), это надо сгладить таймерами/логистикой.
