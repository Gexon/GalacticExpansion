# Глоссарий

**Назначение**: единые определения терминов, чтобы архитектура/код/доки говорили “одним языком”.  
**Аудитория**: разработчики.  
**Статус**: draft.  

## Термины Empyrion/ModAPI

- **Playfield**: игровой “инстанс” (планета, орбита, солнечная орбита и т.д.).
- **Entity**: любая сущность с `entityId` (структура, корабль, NPC и т.п.), управляемая через API.
- **CmdId**: идентификатор типа события/запроса (например `Event_Player_ChangedPlayfield`, `Request_Entity_Spawn`).
- **seqNr**: номер запроса/корреляции. Используется, чтобы сопоставлять ответы на `Game_Request` с приходящими событиями.

## Термины GLEX (проектные)

- **Stage**: стадия развития колонии/объекта (например `LandingPending → ConstructionYard → Base_L1 → ... → Base_Max`).
- **Colony**: логическая сущность симуляции: база + охрана + ресурсные посты + логистика + “план расширения”.
- **World State / state.json**: состояние симуляции, которое переживает перезапуски сервера.

## Классы/типы кораблей (принятое обозначение)

- **SV**: Small Vessel  
- **CV**: Capital Vessel  
- **BA**: Base  
- **HV**: Hover Vessel
