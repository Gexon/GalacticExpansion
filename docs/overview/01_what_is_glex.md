## Коротко

**GalacticExpansion (GLEX)** — server-side DLL мод для Empyrion: Galactic Survival, который делает Zirax “живыми” на захваченной планете:

- встречают игрока патрулями при высадке;
- строят базу по стадиям “площадка → база → улучшения”;
- симуляционно “добывают ресурсы” и расширяются;
- атакуют и мешают высадке игроков;
- реагируют на разрушение объектов игроками (усиление, откат стадии, повторное строительство).

## Целевое окружение

- Dedicated server (server-side мод, без client-side установки).
- Сценарий: ванильный Default Multiplayer.
- Версия игры (по текущему черновику): Empyrion **v1.15 Experimental**.

## Ключевые принципы реализации

- Основные рычаги управления миром доступны через ModAPI: **events** (`Game_Event`) и **requests** (`Game_Request`).
- Стадии колонии реализуются через надёжный паттерн: **destroy старой структуры + spawn новой** (замена prefab’ов).
- “Добыча” и “логистика” в MVP — **симуляция** (не обязана отражать реальные депозиты руды в мире).

## Что НЕ делаем в MVP (non-goals)

- Не делаем client-side UI/скрипты.
- Не делаем полноценный “AI маршрут” кораблей (если ModAPI не даёт управлять маршрутом) — используем “синематик” и таймеры.
- Не пытаемся гарантированно работать по “блокам” (точечный урон/разрушение блоков), если ModAPI работает на уровне сущностей.

## Что считается MVP (первый релиз)

Из `docs/00_наброски проекта.md`:

- 1 “материнская” планета, 1 Zirax база (стадийная).
- Патрули/охранники/строители вокруг этапов строительства.
- “Логистический” корабль появляется над планетой, “доставляет” капсулу, исчезает.
- Конфиг лимитов: максимум баз = 1 на планету, лимиты NPC/дронов/кораблей, интервалы атак.
- Реакция на разрушение: откат/перестройка.
