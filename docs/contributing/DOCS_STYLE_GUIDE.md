# Гайд по стилю документации (GLEX)

**Назначение**: единый стандарт оформления документации, чтобы её было легко поддерживать.  
**Аудитория**: разработчики.  
**Статус**: draft.  

## Шапка документа

В начале каждого документа указываем:

- **Назначение**
- **Аудитория**
- **Статус** (`draft`/`stable`)
- **Связанные документы** (ссылки)

## Структура

Рекомендуемый порядок:

1) Контекст/проблема
2) Решение/подход
3) Детали/алгоритмы
4) Edge cases
5) Примеры

## Диаграммы

Если есть поток requests/events — используем `mermaid` (sequence/flowchart).

## Язык и термины

- Пишем по-русски.
- Термины фиксируем в `docs/overview/02_glossary.md` и не плодим синонимы без надобности.

## Ссылки

- Все ключевые понятия должны иметь “куда дальше” (не оставляем тупиков).
- Внутренние ссылки — относительные, чтобы работали в GitHub/репо.
