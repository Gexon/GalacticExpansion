# Contributing (GLEX)

**Назначение**: правила внесения изменений в проект и документацию.  
**Аудитория**: разработчики.  
**Статус**: draft.  

## Общие принципы

- Изменения должны быть атомарными и объяснимыми (почему, а не только что).
- При добавлении новой механики:
  - обновить архитектурные документы (или добавить ADR),
  - обновить `ConfigReference.md` (если появились новые ключи),
  - добавить/обновить сценарии тестирования.

## Стиль логирования

- Логи должны отвечать на вопросы:
  - “что решили сделать?”,
  - “почему решили?”,
  - “что получилось?”.
- Для requests/events важны: `CmdId`, `seqNr`, результат, ретраи.

## Документация

- Документация — часть продукта: правки в коде без правок в доках допускаются только если изменение тривиально и не влияет на интерфейсы/архитектуру.
- Стиль и шаблон документов — в `DOCS_STYLE_GUIDE.md`.
